#15. 模块化开发

##1.传统非模块化开发有如下的缺点：
-(1)、命名冲突
-(2)、文件依赖

##2.前端标准的模块化规范：
-(1)、AMD - requirejs
-(2)、CMD - seajs

##3.服务器端的模块化规范： CommonJS - Node.js

##4.模块化相关的规则：
-(1)、如何定义模块：一个js文件就是一个模块，模块内部的成员都是相互独立
-(2)、模块成员的导出和引入
—-> 导出模块成员 :
有两种方式：1. exports.sum = sum; —–> 作为一种对象的sum方法导入

                      2.  module.exports = sum;  ----> 将函数sum赋值给...
    ----> 引入模块:   var md = require('./03.js');
          在引入模块时，针对不同的导出方式，js引入的内容也有所不同：
                      对于第一种导出方式 ： 在调用sum 函数时： md.sum(1,2);
                      对于第二种导出方式 ： 在调用sum 函数时： md(1,2);
☆ 注意：模块成员的导出最终以module.exports为准

    如果要导出单个的成员或者比较少的成员，一般我们使用exports导出；
    如果要导出的成员比较多，一般我们使用module.exports的方式
    这两种方式不能同时使用

    exports与module的关系：
    module.exports = exports = {};


    eg： 导出成员：        -------------------------->对应的引入后的用法：var md = require('./03.js');
        exports.sum = sum;      ---------------->      md.sum();
        exports.subtract = subtract;    --------------->  md.subtract();

        module.exports = {
            sum : sum,       --------------->       md.sum();
            subtract : subtract,       --------------->   md.subtract();
            multiply : multiply,    --------------->   md.multiply();
            divide : divide     --------------->   md.divide();
        }

#18.es6 中的 声明变量 let 和 const：
-1. let
+(1). let 声明的变量不存在预解析 ,变量只能先声明再使用
——> eg: console.log(flag); // 报错
let flag = 456;
——> eg: let flag = 456;
console.log(flag); // 456
+(2). let 声明的变量不允许重复（在同一个作用域内）
——> eg: let flag = 123;
let flag = 456;
console.log(flag); // 报错
+(3).ES6引入了块级作用域, 在块内部定义的变量，在外部是不可以访问的 ,在块级作用域内部，变量只能先声明再使用
——> eg: if(true){
let flag = 123;
} // 这是一个块级作用域 ，外部是不可以访问flag的
——> eg: {
let flag = 111;
console.log(flag);
} // 这是一个块级作用域 ，外部是不可以访问flag的
——> eg: for (let i = 0; i < 3; i++) {
// for循环括号中声明的变量只能在循环体中使用
console.log(i);
}
console.log(i);
-2. const
(1).const用来声明常量
(2).const声明的常量不允许重新赋值
(3).const声明的常量必须初始化

-4.字符串的解构赋值
let [a,b,c,d,e,length] = “hello”;

#22.箭头函数

-1.有一个参数时： function foo(v){
                  return v;
                }
                   let foo = v => v;
                            let ret = foo(111);

-2.多个参数必须用小括号包住:
      let foo = (a,b) => {let c = 1; console.log(a + b + c);}
      foo(1,2);
-3.箭头函数的注意事项：
      1、箭头函数中this取决于函数的定义，而不是调用
      function foo(){
          // 使用call调用foo时，这里的this其实就是call的第一个参数
          // console.log(this);
          setTimeout(()=>{
              console.log(this.num);
          },100);
      }
      foo.call({num:1});
      ----------------------------------
      2、箭头函数不可以new
      let foo = () => { this.num = 123;};
      new foo();
      ------------------------------------
      3、箭头函数不可以使用arguments获取参数列表，可以使用rest参数代替
      let foo = (a,b) => {
          // console.log(a,b);
          console.log(arguments);  //这种方式获取不到实参列表
      }
      foo(123,456);

#23. 类与继承
-1. 类： 关键字： class
eg： class Animal{
// 静态方法(静态方法只能通过类名调用，不可以使用实例对象调用) 关键字： static
static showInfo(){
console.log(‘hi’);
}

              // 构造函数
              constructor(name){
                  this.name = name;
              }                      // constructor 只能存属性

              showName(){       // 方法
                  console.log(this.name);
              }
          }
          // let a = new Animal('spike');
          // a.showName();
          // a.showInfo();
          // Animal.showInfo();  ------->  静态方法调用,只能通过类名调用，不可以使用实例对象调用

-2.类的继承   extends
  class Dog extends Animal{
      constructor(name,color){
          super(name);        //super用来调用父类
          this.color = color;
      }

      showColor(){    // 方法
          console.log(this.color);
      }
  }

  let d = new Dog('doudou','yellow');
